# Deep Dive: Data Factory Python Script (`data_factory.py`)

This document provides a line-by-line, engineering-level breakdown of the `data_factory.py` script. 

This script represents the critical transition from **Hardware Setup** to **Data Science**. It acts as the orchestrator, controlling the NgSpice simulator, dynamically altering physical parameters (like the DC Bus voltage), extracting the resulting metrics, and saving them into a dataset for AI training.

---

## 1. Imports and Configuration

```python
import subprocess
import os
import re
import csv
import time
```
* **`subprocess`**: Allows Python to open a hidden terminal and run external applications (NgSpice).
* **`os`**: Interacts with your Windows operating system (handling file paths and checking if files exist).
* **`re`**: Regex (Regular Expressions). An advanced text-scanning engine used to hunt for complex data patterns in massive text logs.
* **`csv`**: Handles reading and writing standard Comma-Separated Values files (the format AI models need to consume data).
* **`time`**: Used to track how long the entire process takes to run.

```python
VOLTAGE_STEPS = range(350, 460, 10)
```
* `range(start, stop, step)` tells Python to generate a list of numbers from 350 up to (but not including) 460, going up by 10.
* **Result:** `[350, 360, 370, 380, 390, 400, 410, 420, 430, 440, 450]`

```python
CIRCUIT_ORIGINAL = os.path.join("src", "H_Bridge_Full.cir")
CIRCUIT_TEMP = os.path.join("src", "H_Bridge_Temp.cir")
CSV_OUTPUT = os.path.join("results", "inverter_dataset.csv")
```
* **`os.path.join`** is a professional way to write folder paths. Instead of hardcoding `"src\H_Bridge...“` (which breaks on Linux/Mac), `os.path.join` automatically uses the correct slashes depending on your operating system.
* We define a `CIRCUIT_TEMP` output file. We *never* want to overwrite your Master `.cir` file, so the script takes a copy, modifies it, runs it, and deletes it.

---

## 2. Regular Expression (Regex) Definitions

```python
meas_pattern = re.compile(r'^\s*(\w+)\s*=\s*([+-]?\d+\.\d+[eE][+-]?\d+)', re.MULTILINE)
```
* This pattern hunts down outputs generated by NgSpice's `.meas` directives.
* `^\s*` = Start at the line edge, ignore blanks.
* `(\w+)` = **Capture Group 1**: Grabs the measurement name (e.g., `v_out_rms`).
* `([+-]?\d+...\d+)` = **Capture Group 2**: Grabs the mathematical value formatted in scientific notation (e.g., `1.92e+02`).

```python
dc_voltage_pattern = re.compile(r'(V_DC_Link\s+400_Bus\s+0\s+)\d+V', re.IGNORECASE)
```
* This pattern hunts exactly for the specific line in your `.cir` file where the DC source is declared.
* It looks for the text `"V_DC_Link   400_Bus   0   "` and captures it in Memory Group 1 using `( )`.
* It then looks for the digits before the `V` (which represents the physical voltage, like `400V`).
* We will use this pattern to erase "400" and inject "350", "360", etc.

---

## 3. The Orchestration Loop

```python
dataset = []
```
* We prepare an empty list. This list will eventually hold 11 different dictionary "folders" (one for each voltage step). At the end, we convert this list into a CSV table.

```python
with open(CIRCUIT_ORIGINAL, "r") as file:
    original_spice_code = file.read()
```
* Python safely opens your Master Circuit file (`"r"` stands for Read mode). 
* `file.read()` copies all 350 lines of SPICE code into memory as one giant string variable (`original_spice_code`).

```python
for v_dc in VOLTAGE_STEPS:
```
* This is the master control loop. Everything indented below this line runs 11 times. During loop 1, `v_dc = 350`. During loop 2, `v_dc = 360`, and so on.

```python
    modified_spice_code = dc_voltage_pattern.sub(rf'\g<1>{v_dc}V', original_spice_code)
```
* **This is the core "Mutation" step.**
* `.sub()` tells Regex to perform a Find-And-Replace.
* It searches `original_spice_code` for the DC voltage line.
* `rf'\g<1>{v_dc}V'` says: *"Keep the exact text you captured in Memory Group 1 (`V_DC_Link 400_Bus 0 `), but immediately append our new {v_dc} variable and a 'V'."*
* As a result, `V_DC_Link 400_Bus 0 400V` is perfectly replaced by `V_DC_Link 400_Bus 0 350V` in the computer's memory.

```python
    with open(CIRCUIT_TEMP, "w") as file:
        file.write(modified_spice_code)
```
* We save the mutated code onto your hard drive as `H_Bridge_Temp.cir` (`"w"` stands for Write mode).

```python
    result = subprocess.run([ngspice, "-b", CIRCUIT_TEMP], capture_output=True, text=True)
```
* Python opens a hidden terminal and runs NgSpice on the Mutated Temporary File in background batch mode (`-b`).
* It waits patiently for the simulation to finish running.

```python
    metrics = {"V_DC_Link": v_dc} 
```
* We prepare an empty Dictionary for this specific simulation step. 
* We intentionally inject `"V_DC_Link": 350` as the very first Key/Value pair so that the final AI dataset tracks exactly what input voltage caused the resulting metrics. *(This is an Independent Variable).*

```python
    for match in meas_pattern.finditer(combined_output):
        name = match.group(1)
        value = float(match.group(2))
        metrics[name] = value
```
* We scan the hidden NgSpice terminal output, extract the Metric names and numerical values, convert them into `float` math numbers, and stuff them inside our `metrics` Dictionary. *(These are the Dependent Variables).*

```python
    dataset.append(metrics)
```
* We drop the entire populated Dictionary chunk into our `dataset` master list.

---

## 4. Finalizing and Saving the File

```python
if os.path.exists(CIRCUIT_TEMP):
    os.remove(CIRCUIT_TEMP)
```
* After all 11 loops finish, we clean up after ourselves by deleting the temporary mutated SPICE file from the hard drive. 

```python
if dataset:
    headers = list(dataset[0].keys())
```
* Python looks at the very first dictionary in our list (the 350V run) and grabs all the Keys (`V_DC_Link`, `v_out_rms`, `p_load_avg`...) to act as the column headers at the top of our CSV file.

```python
    with open(CSV_OUTPUT, mode='w', newline='') as csv_file:
        writer = csv.DictWriter(csv_file, fieldnames=headers)
        writer.writeheader()
        writer.writerows(dataset)
```
* This opens `inverter_dataset.csv` in write mode (`'w'`).
* `csv.DictWriter` is a powerful Python tool designed specifically to convert Python dictionaries directly into Excel-style columns and rows.
* `writer.writeheader()` pastes the Column Titles at the very top.
* `writer.writerows(dataset)` dumps all 11 dictionaries perfectly as rows beneath those columns.
